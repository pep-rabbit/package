# user-cli

Термінальний клієнт (TUI на Textual) для ручного виклику API та перегляду результатів пошуку аптек.

## Архітектурна роль

Модуль є інструментом для розробників і тестового ручного сценарію: дозволяє швидко перевірити API без браузера.

## Відповідальність

- надсилає запити до API за містом і назвою медичної програми/препарату;
- рендерить результати у вигляді карток у терміналі;
- показує помилки HTTP/мережі та порожні відповіді.

## Структура

- `main.py` містить:
	- `Client` (aiohttp-обгортка з semaphore, timeout і керуванням сесією);
	- `SearchApp` (Textual UI, обробка натискань, рендер карток);
	- lifecycle-закриття HTTP-сесії при завершенні застосунку.

## Як працює

1. Користувач вводить місто та програму/препарат.
2. Клієнт викликає `GET /api/top-pharmacies`.
3. Відповідь відображається картками в інтерфейсі Textual.

## Мережева поведінка

- обмеження паралелізму через `asyncio.Semaphore`;
- налаштовані таймаути connect/read/total через `aiohttp.ClientTimeout`;
- уніфікована обробка помилок (`aiohttp`, timeout, JSON decode);
- fallback-відповідь у форматі `{"ok": bool, "status": int, ...}`.

## Бібліотеки та підходи

- `textual` - побудова TUI-інтерфейсу;
- `aiohttp` - асинхронний HTTP-клієнт;
- `asyncio` - керування конкурентністю і життєвим циклом.

Ключовий підхід: мінімальна бізнес-логіка у клієнті, вся доменна логіка залишається на стороні API.

## Конфігурація

- `API_BASE_URL` - базова адреса API.
- значення за замовчуванням: `http://127.0.0.1:8080`.

## Точки розширення

- додати режим CLI-команди (без TUI) з аргументами;
- додати експорт результатів в `json/csv`;
- додати сортування/фільтри в TUI;
- додати інтеграційні тести з mock API.

## Запуск модуля

```bash
API_BASE_URL=http://127.0.0.1:8080 python main.py
```

Якщо API доступне через Nginx, можна вказати:

```bash
API_BASE_URL=http://127.0.0.1 python main.py
```
